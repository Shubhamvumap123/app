## 2024-05-24 - High-Frequency Event Throttling and Memory Reallocation Prevention
**Learning:** High-frequency event listeners (like `scroll` or `resize`) in Client Components can cause performance issues if they directly trigger state updates. Additionally, statically defined reference types (arrays, objects) inside functional components are recreated on every render, which can lead to unnecessary memory allocations and trigger re-renders in child components if passed as props.
**Action:** Always use `requestAnimationFrame` with a `ticking` flag to throttle `scroll` or `resize` handlers in React components to optimize performance and prevent excessive re-renders. Additionally, mark these non-preventable event listeners with `{ passive: true }` to allow immediate browser scrolling without waiting for JavaScript execution. Finally, hoist static data (like configuration arrays or objects) outside of the component definition or wrap them in `useMemo` to prevent unnecessary allocations on each render cycle.
